# Настройка конфигурации сайта МБД

## 🔐 Безопасность конфигурации

Конфигурация сайта хранится в файле `config.json`, который содержит секретные данные (токены, API ключи). **Этот файл добавлен в `.gitignore` и не должен коммититься в Git.**

## 📋 Структура config.json

```json
{
  "telegram": {
    "bot_token": "YOUR_TELEGRAM_BOT_TOKEN",
    "chat_id": "YOUR_TELEGRAM_CHAT_ID"
  },
  "deepseek": {
    "api_key": "YOUR_DEEPSEEK_API_KEY",
    "api_url": "https://api.deepseek.com/v1/chat/completions"
  },
  "security": {
    "rate_limit": {
      "max_requests_per_minute": 10,
      "max_requests_per_hour": 100
    },
    "recaptcha": {
      "site_key": "YOUR_RECAPTCHA_SITE_KEY",
      "secret_key": "YOUR_RECAPTCHA_SECRET_KEY"
    },
    "ddos_protection": {
      "enabled": true,
      "max_concurrent_requests": 50,
      "block_duration_seconds": 300
    }
  }
}
```

## 🚀 Локальная настройка

### 1. Создайте config.json

Файл уже создан в корне проекта. Откройте его и замените плейсхолдеры на реальные значения.

### 2. Получите токен Telegram бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`
3. Следуйте инструкциям для создания бота
4. Сохраните токен (выглядит как `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)
5. Вставьте токен в `config.json` в поле `telegram.bot_token`

### 3. Получите Chat ID

1. Откройте [@userinfobot](https://t.me/userinfobot)
2. Отправьте `/start`
3. Сохраните ваш Chat ID (число)
4. Вставьте Chat ID в `config.json` в поле `telegram.chat_id`

### 4. Получите API ключ DeepSeek

1. Перейдите на [https://platform.deepseek.com/](https://platform.deepseek.com/)
2. Зарегистрируйтесь или войдите
3. Перейдите в раздел API Keys
4. Создайте новый API ключ
5. Сохраните ключ
6. Вставьте ключ в `config.json` в поле `deepseek.api_key`

### 5. (Опционально) Настройте reCAPTCHA

1. Перейдите на [https://www.google.com/recaptcha/admin](https://www.google.com/recaptcha/admin)
2. Зарегистрируйте новый сайт
3. Выберите reCAPTCHA v3
4. Получите Site Key и Secret Key
5. Вставьте их в `config.json` в поля `security.recaptcha.site_key` и `security.recaptcha.secret_key`

## 🌐 Настройка GitHub Pages

### 1. Добавьте Secrets в GitHub

Перейдите в ваш репозиторий на GitHub → Settings → Secrets and variables → Actions → New repository secret

Добавьте следующие секреты:

- `TELEGRAM_BOT_TOKEN` — токен вашего Telegram бота
- `TELEGRAM_CHAT_ID` — ваш Chat ID
- `DEEPSEEK_API_KEY` — API ключ DeepSeek
- `RECAPTCHA_SITE_KEY` — (опционально) Site Key reCAPTCHA
- `RECAPTCHA_SECRET_KEY` — (опционально) Secret Key reCAPTCHA

### 2. Включите GitHub Pages

1. Settings → Pages
2. Source: GitHub Actions
3. Сохраните

### 3. Пушьте изменения

При каждом пуше GitHub Actions автоматически:
1. Создаст `config.json` из Secrets
2. Развернет сайт на GitHub Pages

## 🛡️ Защита от спама и DDoS

### Rate Limiting (Ограничение частоты запросов)

Система автоматически ограничивает количество запросов:

- **Максимум 10 запросов в минуту** — защита от спама в форме заявки
- **Максимум 100 запросов в час** — защита от массовых атак
- **Блокировка на 5 минут** — при превышении лимитов

**Как работает:**
- Каждый запрос (форма заявки, AI-чат) проверяется
- История запросов хранится в памяти браузера
- При превышении лимита пользователь получает уведомление
- После блокировки пользователь должен подождать

### Настройка лимитов

В `config.json` можно изменить:

```json
"security": {
  "rate_limit": {
    "max_requests_per_minute": 10,  // Измените на нужное значение
    "max_requests_per_hour": 100    // Измените на нужное значение
  }
}
```

### Anti-DDoS

Дополнительная защита:

- **Максимум 50 одновременных запросов** — защита от DDoS
- **Автоматическая блокировка** — при подозрительной активности
- **Длительность блокировки** — 300 секунд (5 минут)

**Как работает:**
- Система отслеживает количество активных запросов
- При превышении порога включается блокировка
- Блокировка применяется ко всем запросам с одного IP

## 🔧 Troubleshooting

### Форма заявки не отправляет

1. Проверьте `config.json` — токен и Chat ID должны быть заполнены
2. Проверьте консоль браузера на ошибки
3. Убедитесь, что бот запущен (если используете свой бот)

### AI-ассистент не отвечает

1. Проверьте `config.json` — API ключ DeepSeek должен быть заполнен
2. Проверьте консоль на ошибки
3. Убедитесь, что API ключ действителен
4. Проверьте лимит запросов (10 в минуту)

### Rate limit срабатывает слишком часто

1. Увеличьте лимиты в `config.json`
2. Проверьте, нет ли скриптов, которые делают автоматические запросы
3. Очистите localStorage браузера

### GitHub Actions не создает config.json

1. Проверьте, что все Secrets добавлены
2. Проверьте логи Actions на наличие ошибок
3. Убедитесь, что workflow файл корректен
